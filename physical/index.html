    Physical Entities: patterns¶

    Introduction

    The document outlines modelling choices in relation to physical entities such as physical objects, their parts or their features. The modelling of statements which specify properties, both measured or assigned to the physical object, such as its dimension, or its position in space, are going to be described here. The modelling are provide as CRM patterns as well as in their encoded form using a real-world statements from one of our case studies. The syntax of the encoding is Turtle, and all the prefix used can be found on http://prefix.cc/.

    Features¶

    The physical feature of an object refers to an identifiable sign which are integrally part of the object. A physical feature it is not defined by specific borders (however, it can), and it cannot be detach to the object which carries it.

    The association of an object with its feature in CRM can be modelled as :

    E22 Man-Made Object → P56 bears feature → E26 Physical Feature

    or, if we are talking about man-made feature:

    E22 Man-Made Object → P56 bears feature → E25 Man-made Feature

    watermark and signature¶

    Watermarks and signatures are both typical features of an object, specifically man-made feature, because their identity is not natural but man given.

    Following the modelling above, we can state the fact that an object as a man-made feature as

    E22 Man-Made Object → P56 bears feature → E25 Man-made Feature

    However, for the example to be complete we should make explicit that the object has two features, a signature and a watermark, each of them has a type, and carries a linguistic object. Formally, both they are made explicit as:

    E25 Man-made Feature → P128 carries → E33 Linguistic Object
    E25 Man-made Feature → P2 has type → E55 Type
